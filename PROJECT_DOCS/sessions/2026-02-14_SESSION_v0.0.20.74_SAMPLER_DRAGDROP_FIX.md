## Problem

Samples aus dem Browser konnten nicht in den Pro Audio Sampler gezogen werden. Das Drop-Overlay erschien korrekt, aber das Sample wurde nicht geladen ("No audio loaded" blieb stehen).

**Ursachen:**

1. **Fehlendes `dragMoveEvent`**: Qt's Drag&Drop-Protokoll erfordert, dass ein Widget sowohl `dragEnterEvent` als auch `dragMoveEvent` akzeptiert. Ohne `dragMoveEvent` wird das `dropEvent` nie ausgelöst.

2. **TypeError in `load_wav_file`**: Die Methode `WaveformDisplay.load_wav_file()` in `ui_widgets.py` hatte keinen `engine` Parameter, aber `sampler_widget.py` rief sie mit `engine=self.engine` auf → stiller TypeError im try/except.

```python
def dragMoveEvent(self, e):
    # IMPORTANT: Must accept dragMoveEvent too, otherwise dropEvent won't fire.
    try:
        md = e.mimeData()
        if md and md.hasUrls():
            for u in md.urls():
                ext = Path(u.toLocalFile()).suffix.lower()
                if ext in SUPPORTED_EXTENSIONS:
                    e.acceptProposedAction()
                    return
        e.ignore()
    except Exception:
        try:
            e.ignore()
        except Exception:
            pass
```

## Qt Drag&Drop Erklärung

Qt's Drag&Drop-Protokoll funktioniert so:
1. `dragEnterEvent` → Widget muss `e.acceptProposedAction()` aufrufen
2. `dragMoveEvent` → Widget muss AUCH hier akzeptieren (bei jeder Mausbewegung)
3. `dropEvent` → Wird nur aufgerufen, wenn beide vorherigen Events akzeptiert wurden

Ohne `dragMoveEvent` ignoriert Qt das Drop stillschweigend.

## Ziel dieser Session
Tie (Haltebogen) soll **echt** wirken: gebundene Noten dürfen beim Playback nicht erneut anschlagen, sondern müssen als eine lange Note gerendert werden.

## Umsetzung (Playback-Logik)
- Ties sind bereits als Marks gespeichert:
  - `Project.notation_marks` mit `type="tie"` und `data.from/to` (beat + pitch)
- In der Audio-Render-Pipeline werden diese Marks nun ausgewertet:
  - Pro Pitch werden Tie-Chains (A→B→C …) gebildet
  - Noten werden **non-destructive** gemerged:
    - Erste Note wird verlängert bis zum Ende der letzten Note der Chain
    - Folge-Noten werden für das Rendering entfernt, damit kein Re-Trigger entsteht

## Technische Details
- Implementiert in `pydaw/audio/arrangement_renderer.py` als `_apply_ties_to_notes()`
- Unterstützt:
  - Notes als Model-Objekte **oder** Dicts (Backward Compatibility)
  - Mehrere Chains pro Pitch
  - Defensive Loop-Guards (max 128 Schritte pro Chain)
- Cache/Render:
  - Content-Hash basiert auf den **gemergten** Notes → Tie-Änderungen invalidieren automatisch den Render-Cache
- Zusätzlich auch im Sounddevice-Arrangement-Pfad aktiviert:
  - `pydaw/audio/audio_engine.py` nutzt `_apply_ties_to_notes()` ebenfalls vor dem Rendern

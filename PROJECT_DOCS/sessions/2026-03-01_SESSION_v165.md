## Ziel / User-Report

1) **Arranger:** Lasso/Mehrfachauswahl von Audio-Clips → **Ctrl+J / Consolidate** soll funktionieren (wie MIDI), aktuell „passiert nichts“.
2) **Reverse Workflow:**
   - Im **Arranger** soll ein Audio-Clip per Klick + Rechtsklick-Menü **Reverse** gedreht werden können.
   - Der Clip muss den Reverse-Zustand **sichtbar** anzeigen.
   - Im **Audio Editor** soll „Reverse“ auch funktionieren, wenn **kein Event selektiert** ist (typisch: man klickt den Clip im Arranger, öffnet Editor, Rechtsklick → Reverse).

Wichtig: **nichts kaputt machen** und bestehende Clip-Editor Workflows (Ctrl+J etc.) behalten.

---

## Analyse (Root Cause)

### A) Arranger Ctrl+J
- `ArrangerKeyboardHandler._join_clips()` hatte bereits Audio-Bounce-Consolidate.
- Aber: `ArrangerCanvas.keyPressEvent()` und Kontext-Menü nutzten noch den **Legacy-Pfad** `project.join_clips()`.
- Ergebnis: MIDI konnte joinen, Audio-Consolidate wurde nicht zuverlässig ausgeführt (Errors wurden außerdem defensiv geschluckt).

### B) Reverse im Editor
- Audio-Editor Kontext-Menü „Reverse“ war an **Event-Selektion** gebunden.
- Wenn User auf den Clip klickt (Arranger-Selektion) und im Editor im Hintergrund rechtsklickt, ist **keine Event-Selektion aktiv** → Reverse war wirkungslos.

---

## Umsetzung (Safe Fix)

### 1) Arranger: Ctrl+J wirklich über KeyboardHandler
- `ArrangerCanvas.keyPressEvent()` delegiert Ctrl+J jetzt an den zentralen `ArrangerKeyboardHandler._join_clips()`.
- Dadurch funktioniert:
  - **MIDI Join** wie vorher.
  - **Audio-Consolidate (Bounce)** zuverlässig.

### 2) Arranger Kontext-Menü: Multi-Select bleibt erhalten
- Rechtsklick auf ein bereits selektiertes Clip-Item **behält die Mehrfachauswahl**.
- Nur wenn auf einen **nicht selektierten** Clip rechtsgeklickt wird, wird die Auswahl auf diesen Clip reduziert.

### 3) Arranger Kontext-Menü: Reverse + Consolidate (Keep Originals)
- Für Audio-Selektion (einzeln oder multi): **Reverse** als clip-level Toggle.
- Für multi-selection (gleiche Spur + gleicher Typ):
  - **Consolidate (Ctrl+J)**
  - **Consolidate (Originale behalten)** (non-destructive)

### 4) Audio Editor: Reverse = Events ODER Clip
- „Reverse“ bleibt **per-event**, wenn Events selektiert sind.
- Wenn **keine Event-Selektion** existiert, toggelt „Reverse“ automatisch den **Clip-Reverse** (`clip.reversed`).
- Ergebnis: 1 Clip auswählen → Editor → Rechtsklick → Reverse funktioniert sofort.

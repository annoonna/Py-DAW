# Session Log — v0.0.20.110 — Bachs Orgel UI Lesbarkeit im DevicePanel (Breiten-Fix)

**Datum:** 2026-02-22  
**Bearbeiter:** GPT-5.2 Thinking (ChatGPT)
**Typ:** UI-only / sicher (keine Audio-/DSP-Änderungen)

## Ausgangsproblem (User-Feedback)
In der Device-Ansicht waren Teile des Instruments **Bachs Orgel** schlecht lesbar bzw. abgeschnitten:
- Knob-Labels/Parametertexte wurden zusammengequetscht
- Drawbar-/Stop-Bereich wirkte zu eng
- Viel freie Fläche rechts, aber Instrument-Card blieb zu schmal

## Ursache
Die Instrument-Card erhielt eine zu kleine Breite (generische DeviceCard-MinWidth + komprimierbare Layouts im Instrument-Widget).
Dadurch wurde das Bach-Orgel-Widget intern horizontal zusammengedrückt.

## Umsetzung (UI-only)
### 1) Bachs-Orgel-Widget fordert realistische Breite an
- `minimumSizeHint()` + `sizeHint()` ergänzt
- Mindestbreiten pro UI-Sektion (Tempo/Synthesis/Control/Effects/Bachs Orgel/I-O) gesetzt
- Komboboxen/Preset-Combo mit Mindestbreiten versehen
- Top-Layout ohne Quetsch-Stretch für Sektionen (freiwilliger Restplatz via `addStretch`)

### 2) DevicePanel übernimmt Instrument-Breiten-Hint gezielt
- In der Instrument-Card-Erzeugung wird die gewünschte Breite des Instrument-Widgets aus `minimumSizeHint()/sizeHint()` gelesen
- Card-MinWidth wird entsprechend erhöht (mit Obergrenze), damit Instrumente wie Bachs Orgel lesbar bleiben
- Keine Änderung an FX-Reorder/DnD-Logik, nur Darstellung

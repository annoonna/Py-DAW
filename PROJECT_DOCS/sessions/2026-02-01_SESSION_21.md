## Kontext
User-Report:
- App startet nicht mehr: „Unhandled Python exception“ → PyQt6 SIGABRT direkt beim Start.
- In den Audio-Einstellungen waren JACK/Port-Optionen teils ausgegraut / Backend wurde unerwartet überschrieben.

## Ursache
`pydaw/ui/audio_settings_dialog.py` hatte durch ein Indentation-Regression mehrere Methoden (`_on_backend_changed`, `_refresh_lists`, `accept`, `_start_audio`) **außerhalb** der Klasse definiert.
Beim Connect/Call im UI entstand dadurch eine Exception, die in PyQt6 als „Unhandled Python exception“ endet und den Prozess abbricht (SIGABRT).

## Fixes
- AudioSettingsDialog komplett bereinigt (alle Methoden korrekt als Klassenmethoden).
- Backend-Wahl bleibt stabil:
  - „JACK Client aktivieren“ schaltet **nicht** mehr das Engine-Backend um.
  - Sounddevice Input/Output IDs werden nur gespeichert, wenn Backend=sounddevice.
- JACK Port-Controls bleiben nutzbar **unabhängig** vom Backend (nur Port-Visibility/Routing).
- Services: JACK client kann (wenn aktiviert) unabhängig vom Engine-Backend gestartet werden (qpwgraph Sichtbarkeit).

## Geänderte Dateien
- `pydaw/ui/audio_settings_dialog.py` (Crashfix + UX-Regeln)
- `pydaw/app.py` (SafeApplication + JACK prefs early)

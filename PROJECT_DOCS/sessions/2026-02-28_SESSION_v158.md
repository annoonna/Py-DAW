## Umsetzung (safe)

### A) Transport → Audio Editor Playhead
- AudioEditor hängt jetzt an `transport.playhead_changed`.
- Im Audio Editor wird **lokaler Clip-Playhead** gezeichnet (Loop-Wrap), statt globaler Timeline-Beat.
- Nach `refresh()` wird die Playhead-Linie wieder angelegt (Scene-Rebuild safe).

### B) Loop zeichnen ohne Tool-Wechsel
- **Alt+Drag im Editor** setzt/zieht die Clip-Loop-Region direkt.
- **Alt+Drag im Ruler** setzt/zieht die Loop-Region ebenfalls.
- Tool-Label wurde klarer: **Loop** statt "Time".

### C) Lupe/Zoom wie Bitwig
- Neues Tool: **Lupe** (ZOOM) im Tool-Dropdown.
- Click+Drag **hoch/runter** zoomt horizontal (AnchorUnderMouse).
- Zusätzlich: Beat-Ruler unterstützt Zoom-Geste (vertikales Ziehen = Zoom).

### D) Zeiger = DAW-Pointer
- Event-Blöcke (Audio Events) akzeptieren jetzt **POINTER oder ARROW**.
- Dadurch funktionieren im Zeiger-Tool wie erwartet:
  - Selektieren
  - Drag-Move
  - Alt-Drag Duplicate
  - Ctrl+C/V/D Workflows

### E) Waveform-Tiling korrekt
- Waveform-Mapping wurde von "Beat->Fraction->Samples" auf **Beat→Sekunden→Samples** umgestellt:
  - nutzt Projekt-BPM + Clip `offset_seconds` + `stretch`
  - Loop-Wrap zeigt echte Wiederholung (kein Stretch-to-fit)

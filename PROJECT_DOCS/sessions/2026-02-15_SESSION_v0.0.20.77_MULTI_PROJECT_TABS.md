## ARCHITEKTUR-ENTSCHEIDUNGEN

### Nur EINE Audio-Engine aktiv
- Beim Tab-Wechsel: Transport stopp → alten Projekt-Context sichern → neuen binden
- Audio-Engine wird per `bind_transport()` an neues Projekt rebindet
- BPM/TS/Loop werden vom neuen Projekt synchronisiert

### Vollständig modular (nichts kaputt)
- 4 neue Dateien, 0 gelöschte
- Bestehende MainWindow-Methoden unverändert
- ProjectTabService ist optional (graceful fallback wenn nicht vorhanden)
- Alle neuen UI-Elemente in try/except (crash-sicher)

### Cross-Project Drag&Drop
- Custom MIME-Type: `application/x-pydaw-cross-project`
- JSON-Payload mit Source-Tab-Index, Track/Clip-IDs
- Deep-Copy mit neuen IDs (kein ID-Conflict)
- Media-Referenzen werden mitgenommen

## Ziel / User-Report

1) **Clip Launcher:** Klick auf Slot darf **nicht** sofort abspielen (kein Transport-Start, kein Launch).  
   Stattdessen: **auswählen + Editor öffnen**. Launch nur über einen kleinen ▶-Button im Slot.

2) **Audio Editor:** Bei gesetzter Clip-Loop-Region muss die Waveform im Editor **sichtbar wiederholt** werden
   (tiled Loop) – wie im Referenz-Screenshot.

---

## Umsetzung (safe)

### A) Clip Launcher (pydaw/ui/clip_launcher.py)

- `_slot_clicked()` umgebaut:
  - macht jetzt **Selection + Editor-Fokus** (emit `clip_activated`)  
  - **kein** `launcher.launch_slot()` mehr auf Single-Click  
- `SlotWaveButton` erweitert:
  - zeichnet einen kleinen **▶-Button (oben rechts)**  
  - Klick in die Hotzone startet Launch via `_panel._launch(slot_key)`  
  - unterdrückt danach das normale `QPushButton.clicked`, damit kein Double-Action passiert

### B) Audio Editor Loop-Tiling (pydaw/ui/audio_editor/audio_event_editor.py)

- `_draw_waveform(...)` erweitert um Loop-Parameter:
  - mappt Display-Beat → (Loop-Wrap) → Source-Sample → Peak-Table
  - Ergebnis: **Loop-Region wiederholt sich optisch** über die gesamte Clip-Länge

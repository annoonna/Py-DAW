# Session — 2026-02-05 — v0.0.19.7.41

## ✅ Ziel (User-Request)
Drag & Drop von Audio-Dateien aus dem Browser **in den Clip Launcher**: 
- Drop auf das Overlay erzeugt/füllt einen **Clip-Launcher Slot** (Scene x Track)
- **kein** Clip im Arranger (Hintergrund)
- Slot zeigt **Dateiname + echte Waveform-Vorschau** (Pro-DAW-like, wie im Arranger)

## 🧩 Implementiert
### 1) Overlay-Drop-Logik final
- Overlay nutzt nun **Scene-Index 1..N** (konsistent zum Clip Launcher Grid)
- Overlay **acceptProposedAction()** auch dann, wenn nicht exakt auf einem Slot gedroppt wird → blockiert Arranger zuverlässig
- Wenn noch keine Tracks existieren: Overlay erzeugt automatisch **Audio Track** via `ProjectService.ensure_audio_track()`

### 2) Backend: place_in_arranger wird respektiert
- `ProjectService.add_audio_clip_from_file_at(... place_in_arranger=False)` setzt nun:
  - `clip.launcher_only = True`
- Arranger blendet `launcher_only` Clips aus (`ArrangerCanvas._arranger_clips`)

### 3) Clip Launcher Slots: Waveform Preview
- Slots verwenden jetzt `SlotWaveButton` (Custom Paint)
- Waveform-Peaks werden **cached** und asynchron berechnet (ähnlich Arranger)
- Darstellung: Name oben, darunter Waveform-Linien

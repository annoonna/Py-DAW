# 📝 SESSION LOG: 2026-02-22 — Bitwig/Ableton-Style Projekt-Speicherung

**Entwickler:** Claude Opus 4.6 (Anthropic)
**Version:** v0.0.20.130 → v0.0.20.131
**Task:** CRITICAL — Sampler/Instrumente werden beim Laden nicht wiederhergestellt

## PROBLEM (User-Report)

Beim Speichern und Wiederladen eines Projekts werden Sampler, Drum Machine Samples,
SF2-Instrumente und alle zugehörigen Einstellungen NICHT wiederhergestellt.
Der User wünscht sich ein Bitwig/Ableton-Style System:
- Kompletter Projektordner mit ALLEN Samples, Instrumenten, Presets
- Beim Laden alles automatisch wiederherstellen

## ROOT CAUSE ANALYSE

### Bug 1: Fehlende `import_audio_to_project()` Methode (KRITISCH)
- Sampler (`sampler_widget.py:520`) und DrumMachine (`drum_widget.py:298`) rufen
  `self.project_service.import_audio_to_project(track_id, path, label=...)` auf
- Diese Methode **existierte NICHT** auf `ProjectService`!
- Der try/except schluckte den `AttributeError` still → Samples wurden NIE
  in den Projektordner (media/) importiert → `media_id` blieb immer leer
- Folge: Absolute Pfade gespeichert → nach Verschieben/Reload = Dateien nicht gefunden

### Bug 2: `_package_project_media()` ignorierte Instrument-State
- Beim Speichern wurden nur `project.media` und `project.clips` verarbeitet
- Sample-Pfade in `track.instrument_state` (Sampler, DrumMachine) wurden
  NICHT ins Projekt kopiert und NICHT relativiert
- SF2-Pfade auf Tracks wurden ebenfalls nicht verarbeitet

### Bug 3: `_resolve_project_paths_after_load()` ignorierte Instrument-State
- Beim Laden wurden relative Pfade in `track.instrument_state` NICHT
  zurück in absolute Pfade aufgelöst
- Auch SF2-Pfade wurden nicht aufgelöst

## FIX (3 Kernmaßnahmen)

### 1. Neue Methode: `ProjectService.import_audio_to_project()`
```python
def import_audio_to_project(self, track_id, path, label=""):
    """Bitwig/Ableton-style: Sample ins <project>/media/ importieren."""
    item = fm_import_audio(path, self.ctx)
    return (str(item.path), str(item.id))
```
→ Sampler + DrumMachine können jetzt korrekt importieren

### 2. `_package_project_media()` erweitert
- SF2-Dateien: Werden ins `media/` kopiert, Pfad wird relativ
- `track.instrument_state`: Alle Sample-Pfade (Sampler, DrumMachine Slots)
  werden ins `media/` kopiert und relativiert
- Neue Helper: `_package_instrument_sample_path()`

### 3. `_resolve_project_paths_after_load()` + `resolve_loaded_media_paths()` erweitert
- SF2-Pfade auf Tracks werden aufgelöst
- Instrument-State Sample-Pfade werden aufgelöst
- Neue Helpers: `_resolve_instrument_state_paths()`, `_resolve_state_path()`

## Ziel
Performance verbessern bei SF2/FluidSynth & schnellem Workflow: MIDI-Clips sollen beim Laden/optional vor Play im Hintergrund in den bestehenden WAV-Cache gerendert werden ("Ready for Bach").

## Änderungen
### 1) MIDI Pre-Render (Background)
- Neuer Background-Job in `ProjectService`: sammelt MIDI-Clips, wendet Tie-Merge an (wie Playback), rendert WAVs über `ensure_rendered_wav`.
- Progress-Signale:
  - `prerender_started(total)`
  - `prerender_label(text)`
  - `prerender_progress(percent)`
  - `prerender_finished(ok)`
- Cancel: UI setzt Cancel-Flag; Job bricht nach aktuellem Clip ab.

### 2) UI Integration
- Audio-Menü: **"MIDI-Clips vorbereiten (Pre-Render)"** (Ctrl+Alt+R)
- Auto-Start nach Projekt öffnen / Snapshot laden (silent, ohne Dialog).
- Play-Button: wenn Pre-Render läuft → wartet kurz; optional Start von Pre-Render vor Play.
- Statusbar zeigt kurze Meldungen, Progressdialog nur wenn vom User ausgelöst oder vor Play.
